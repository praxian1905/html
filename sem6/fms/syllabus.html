Unit 02

02. Automated Assembly Systems

🔩 Unit Overview

Covers the design principles that make products amenable to automation, the machine architectures used for high-speed assembly, part feeding and delivery mechanisms, and the quantitative models used to evaluate system performance.

Design for Automated Assembly (DFAA)

  • Reducing part count — minimise total number of components
  • Design parts for easy orientation and feeding
  • Symmetrical parts preferred; avoid tangling/nesting
  • Design for ease of insertion — chamfers, leads, self-locating features
  • Avoid flexible, sticky, or abrasive parts where possible

Types of Automated Assembly Systems

Type 01 ★

Rotary Dial (Indexing) Machine

  • Circular worktable with fixed stations
  • Synchronous indexing between stations
  • High speed, small parts
Type 02

In-Line Transfer Machine

  • Linear sequence of assembly stations
  • Suitable for larger components
  • Synchronous or asynchronous transport
Type 03

Carousel / Loop System

  • Closed-loop conveyor with pallets
  • Asynchronous — each pallet independent
  • Flexible, easy to reconfigure
Type 04

Single-Station Machine

  • Robot or multi-arm unit at one station
  • Full assembly at one location
  • Low volume, high complexity

Part Feeding Devices

  • Vibratory bowl feeder — most common; orients parts via track geometry
  • Rotary disc feeder, blade feeder, reciprocating tube feeder
  • Centrifugal hopper feeder — high feed rates for small parts
  • Feeder efficiency: feed rate, jam frequency, orientation quality

Quantitative Analysis

  • Delivery system operation — feed rate, efficiency, proportion of acceptable parts
  • Analysis of a single-station assembly machine
  • Production rate, yield, proportion defective
  • Effect of defective parts on machine jams and downtime
Single-Station Assembly Production Rate: Rp = 1 / Tc·(1 + mθpd·Td / Tc) where Tc = assembly cycle time m = number of part additions θ = probability part causes jam pd = fraction defective Td = mean downtime per jam
Unit 03

03. Group Technology

🗂️ Unit Overview

Introduces the philosophy of exploiting design and manufacturing similarities among parts to improve efficiency — covering part family identification through classification and coding, cellular manufacturing, rank-order clustering, and the rapid-changeover discipline of SMED.

Part Families & Classification

  • Part families — groups of parts similar in design or manufacturing attributes
  • Methods of forming families: visual inspection, production flow analysis, coding
  • Parts classification and coding systems
  • Monocode (hierarchical / tree structure)
  • Polycode (chain / attribute structure)
  • Mixed / hybrid code
  • Opitz Parts Classification & Coding System
  • Form code (5 digits): overall shape, external/internal shape, plane surface, auxiliary holes & gear teeth
  • Supplementary code (4 digits): dimensions, material, starting blank, accuracy
Opitz Code Structure: | 1 | 2 | 3 | 4 | 5 || 6 | 7 | 8 | 9 | ↑___FORM CODE_____↑ ↑_SUPPLEMENTARY_↑ Digit 1: Part class (rotational L/D ≤ 0.5 / ≤ 3 / > 3, non-rotational) Digit 2: External shape and shape elements Digit 3: Internal shape Digit 4: Plane surface machining Digit 5: Auxiliary holes, gear teeth, forming

Production Flow Analysis & Cellular Manufacturing

  • Production flow analysis (PFA) — identifies part families from route cards
  • Composite part concept — hypothetical part embodying all features of a family
  • Types of machine cells and layouts
  • Single machine cell
  • Group machine cell with GT layout
  • Flexible manufacturing cell (FMC)
  • Rank Order Clustering (ROC) — binary matrix algorithm to group parts and machines
  • Assign binary weights to rows/columns → rank → rearrange → iterate
  • Applications of group technology — reduced setup, WIP, lead time; improved quality

Single-Minute Exchange of Die (SMED)

SMED 01

Observe & Record

Video the entire current changeover without interruption.

SMED 02

Separate Internal / External

Identify which tasks must be done with machine stopped (internal) vs. running (external).

SMED 03

Convert Internal → External

Pre-stage tools, fixtures, and materials before shutdown.

SMED 04

Streamline All Activities

Standardise fasteners, parallel tasks, functional clamps — target < 10 min.

💡 SMED Goal

Reduce every changeover to single-digit minutes (under 10 min), enabling smaller batch sizes, reduced inventory, and faster response to demand changes.

04. Flexible Manufacturing Systems

🏭 Unit Overview

Examines the complete architecture of an FMS — its constituent workstations, material handling and storage infrastructure, layout options, computer control hierarchy, and the industrial contexts in which FMS delivers the greatest benefit.

FMS Components & Types

FMS Type 01 ★

Flexible Manufacturing Cell

  • 2–3 CNC machines + robot
  • Automated pallet changer
  • Small part family, high flexibility
FMS Type 02

Flexible Manufacturing Group

  • 4–6 machines, shared transport
  • Part families with moderate variety
  • Central tool storage
FMS Type 03

Flexible Manufacturing Line

  • Linear or loop layout
  • High volume, moderate variety
  • Dedicated AGV or conveyor
FMS Type 04 ★

Flexible Transfer Line

  • Quasi-dedicated; limited variety
  • CNC stations replace fixed heads
  • Bridges gap to Detroit automation

FMS Workstations

  • CNC machining centres — horizontal and vertical spindle
  • CNC turning centres and mill-turn machines
  • Inspection and coordinate measuring stations
  • Washing / cleaning stations
  • Load / unload stations — manual or robotic

Material Handling & Storage

  • Functions of the handling system — transport, buffer, tool delivery, swarf removal
  • FMS layout configurations
  • In-line layout — linear AGV path
  • Loop layout — uni-directional closed loop
  • Ladder layout — rungs between two rails
  • Open field layout — maximum flexibility, complex routing
  • Material handling equipment
  • Automated Guided Vehicles (AGVs) — wire-guided, laser, vision
  • Rail-guided vehicles (RGVs)
  • Roller conveyors and pallet transfer systems
  • Industrial robots for machine tending
  • Computer control system
  • Workstation control, cell control, FMS supervisory control
  • DNC — direct/distributed numerical control
  • Tool management, part scheduling, monitoring

Unit 05

05. Robotic Technology

🦾 Unit Overview

Provides a thorough grounding in industrial robot mechanics and specifications — from joint and link anatomy through configuration geometries and work volumes to control types, end effectors, sensors, and application domains.

Robot Anatomy — Joints & Links

  • Joints and links — revolute (R), prismatic (P), cylindrical (C), spherical (S) joints
  • Degrees of freedom (DOF) — typically 4 to 6 for industrial robots
  • Manipulator arm, wrist assembly, end-of-arm tooling (EOAT)

Common Robot Configurations

📐

CARTESIAN (PPP)

Three linear axes; simple kinematics; high positional accuracy.

🔄

CYLINDRICAL (RPP)

Rotary base + 2 linear axes; cylindrical work volume.

🌐

SPHERICAL (RRP)

Polar / spherical envelope; used in welding & spray painting.

🦾

SCARA (RRP)

Selective compliance; fast, ideal for pick-and-place assembly.

🤖

ARTICULATED (RRR)

6-DOF jointed arm; most versatile; dominant in industry.

⬡

PARALLEL / DELTA

High speed, high stiffness; food & pharmaceutical pick-and-place.

Specifications & Performance

  • Work volume — 3D envelope reachable by robot wrist; depends on configuration
  • Types of robot control
  • Limited-sequence (bang-bang) — end-stop positioning
  • Point-to-point (PTP) — positions stored, paths not controlled
  • Continuous path (CP) — full path and velocity control
  • Intelligent / adaptive control — sensor-guided, vision-based
  • Accuracy — deviation of achieved pose from commanded pose
  • Repeatability — consistency of returning to a taught point (typically ±0.02–0.1 mm)
  • Other specifications — payload, reach, speed, IP rating, cycle time

End Effectors & Sensors

  • End effectors (EOAT)
  • Grippers — two-finger, parallel, angular, vacuum, magnetic
  • Process tools — welding gun, spray nozzle, drill spindle, sealing gun
  • Sensors in robotics
  • Internal: encoders, resolvers, tachometers (joint feedback)
  • External: force/torque sensors, proximity, vision systems, laser rangefinders
  • Robot applications — spot/arc welding, assembly, painting, palletising, machine tending, inspection

Unit 06

06. Robot Programming

💻 Unit Overview

Covers the full spectrum of robot programming methods — from manual lead-through teaching to textual motion languages, off-line simulation, and work-cell supervisory control — including the interlocking mechanisms that ensure safe coordinated operation.

Types of Programming

Method 01 ★

Lead-Through Programming

  • Operator physically guides robot arm
  • Poses or path recorded automatically
  • Fastest for painting, sealing paths
Method 02

Teach-Pendant (Walk-Through)

  • Jogging robot joint by joint
  • Points saved to program memory
  • Common for welding, pick-and-place
Method 03

Textual / Language Programming

  • Written in robot language (VAL, RAPID, KRL)
  • Supports logic, loops, sensor branching
  • Complex assembly, adaptive tasks
Method 04

Off-Line Programming (OLP)

  • Programmed in simulation software
  • No robot downtime during programming
  • Path optimisation before deployment

Motion Programming

  • Motion types
  • Joint interpolation (MOVEJ) — fastest, unconstrained path
  • Linear interpolation (MOVEL) — TCP moves in straight line
  • Circular interpolation (MOVEC) — arc path defined by three points
  • Speed and zone (blending) parameters — smooth transitions between points
  • Payload and inertia declarations for dynamic model compensation

Interlocks

  • Hardware interlocks — safety fences, light curtains, area scanners
  • Software interlocks — I/O handshaking signals between robot and peripheral equipment
  • Wait-for-input / set-output instructions in program
  • Coordinated motion interlocks — synchronisation with conveyors, positioners, other robots
  • Emergency stop and safeguard monitoring — Category 0/1/2 stops per ISO 10218

Simulation, OLP & Work Cell Control

  • Simulation and off-line programming
  • Tools: RoboDK, ABB RobotStudio, KUKA Sim, Delmia

  • Work cell control

  • Communication: Profinet, EtherNet/IP, OPC-UA for Industry 4.0 integration

Advantages & Disadvantages

Advantages

Reasons to Automate Robotically

  • 24/7 operation — no fatigue
  • Consistent quality, high repeatability
  • Safe in hazardous environments
  • Reprogrammable for new tasks
  • Integration with vision & sensors
Disadvantages

Limitations & Challenges

  • High initial capital investment
  • Requires skilled programming personnel
  • Limited adaptability to unstructured tasks
  • Safety fencing increases floor space
  • Maintenance and downtime costs
Off-line programming and digital twin simulation are rapidly converging — modern FMS cells are virtually commissioned in software before a single bolt is tightened on the shop floor, cutting installation time and risk dramatically.

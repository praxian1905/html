01. Basics of CAD

🖥️ Unit Overview

Establishes the conceptual and mathematical foundations of Computer Aided Design — from the scope and benefits of CAD over manual drafting, through coordinate systems and 3D space, to the matrix algebra of 2D and 3D geometric transformations that underpin every CAD kernel.

Introduction to CAD

  • Need and scope of Computer Aided Design — product lifecycle, design iteration speed
  • Fundamentals of CAD and computer graphics — display hardware, input devices, graphics pipeline
  • Application areas — mechanical, civil, architectural, VLSI, aerospace design
  • Comparison of CAD with manual designing — accuracy, speed, reuse, documentation
  • Benefits of CAD — reduced errors, faster prototyping, integration with CAM/CAE
  • User Coordinate System (UCS) and World Coordinate System (WCS)
  • 2D and 3D coordinate systems; right-hand rule; homogeneous coordinates

Unit 02

02. Curves and Surfaces

📈 Unit Overview

Covers the mathematical representations used to define smooth, precise curves and surfaces in CAD systems — from analytic conics through parametric synthetic curves (cubic spline, Bézier, B-spline, NURBS) to the surface types critical in automotive and industrial design.

Analytic Curves

  • Representation of circle — implicit, parametric (cos t, sin t); arc definition
  • Ellipse — semi-axes a, b; parametric form (a·cos t, b·sin t)
  • Parabola — reflector geometry; parametric and standard forms
  • Hyperbola — two-branch curve; applications in gear tooth profiles

Curve 01 ★

Cubic Spline

  • Piecewise cubic polynomial; C² continuity
  • Equation: P(t) = a₀ + a₁t + a₂t² + a₃t³
  • Natural, clamped, periodic end conditions
  • Properties: minimum curvature (stiffest spline)
Curve 02 ★

Bézier Curve

  • P(t) = Σ Bᵢ Cᵢⁿ tⁱ (1−t)ⁿ⁻ⁱ (Bernstein basis)
  • Convex hull, endpoint interpolation
  • Variation-diminishing; affine invariance
  • Degree = number of control points − 1
Curve 03 ★

B-Spline

  • Local control — moving one point affects only k spans
  • Degree independent of control polygon count
  • Uniform, non-uniform, open, closed types
  • Generalises Bézier; C^(k-2) continuity
Curve 04 ★

NURBS

  • Non-Uniform Rational B-Splines
  • Exact representation of conics (circle, ellipse)
  • Weights modify curve toward control points
  • Industry standard (IGES, STEP, ACIS, Parasolid)

Continuity & Surfaces

  • Concept of continuity — G⁰ (positional), G¹ (tangent), G² (curvature); C⁰, C¹, C² (parametric)
  • Types of surfaces used in the automotive industry
  • Class-A surfaces — curvature-continuous, visible exterior body panels
  • Bilinear, ruled, and lofted surfaces
  • Bézier and B-spline / NURBS surface patches
  • Coons patch — blending four boundary curves
  • Swept and revolved surfaces; fillet and blend surfaces
  • Applications — hood, door panel, windscreen, turbine blade, car bumper
Bézier Curve (degree n): P(t) = Σᵢ₌₀ⁿ Bᵢ · Cᵢⁿ · tⁱ · (1-t)^(n-i) t ∈ [0,1] where Cᵢⁿ = n! / (i!(n-i)!) (binomial coefficient) Bᵢ = control points NURBS Curve (rational B-spline): C(u) = Σ Nᵢ,ₖ(u) · wᵢ · Pᵢ / Σ Nᵢ,ₖ(u) · wᵢ Nᵢ,ₖ = B-spline basis functions of degree k wᵢ = weights (wᵢ = 1 → non-rational = B-spline)
Unit 03

03. Solid Modelling

🧊 Unit Overview

Introduces the rigorous geometric and topological framework of solid modelling — comparing wireframe, surface, and solid representations — and surveys the principal solid modelling schemes (B-rep, CSG, sweep, primitive instancing, cell decomposition) along with Boolean operations that combine them.

Geometry vs. Topology & Model Comparison

Wireframe Model

  • Edges and vertices only
  • No surface or volume data
  • Ambiguous — multiple interpretations
  • Fast; used in early CAD
  • No mass properties, no FEA

Surface Model

  • Edges + faces; no enclosed volume
  • Defines shape but not interior
  • Used in styling (Class-A)
  • Rendering and toolpath possible
  • No volume / mass properties

Solid Model ★

  • Complete: vertices + edges + faces + volume
  • Unambiguous; physically valid
  • Mass, inertia, centroid computable
  • Supports FEA, interference check
  • Industry standard for CAD/CAM/CAE

Solid Modelling Concepts

  • Geometry (shape — curves, surfaces) vs. Topology (connectivity — V, E, F relationships)
  • Euler–Poincaré formula: V − E + F − H = 2(B − G)   (validity check)
  • Properties of solid models — closure, rigidity, finite, orientability
  • Properties of representation schemes — domain, unambiguity, uniqueness, conciseness, efficiency
  • Concept of half-spaces — a solid defined as intersection of half-spaces (S = H₁ ∩ H₂ ∩ … ∩ Hₙ)
  • Boolean operations — Union (∪), Intersection (∩), Difference (−); regularised Boolean

Representation Schemes

Scheme 01 ★

B-rep (Boundary Representation)

  • Solid described by bounding faces, edges, vertices
  • Data structure: face–loop–edge–vertex (half-edge)
  • Basis of most commercial CAD kernels (ACIS, Parasolid)
  • Excellent for visualisation and NC toolpath
Scheme 02 ★

CSG (Constructive Solid Geometry)

  • Binary tree of Boolean ops on primitives
  • Compact; easy to edit; history-based
  • Feature-based modelling (Pro-E, SolidWorks)
  • Evaluation needed for rendering
Scheme 03

Sweep Representation

  • Extrude or revolve a 2D profile
  • Translational and rotational sweep
  • Generalised sweep along arbitrary spine
Scheme 04

Primitive Instancing

  • Library of parameterised primitives
  • Bolt, hole, gear families
  • Efficient for standard components
Scheme 05

Cell Decomposition

  • Solid partitioned into non-overlapping cells
  • Spatial occupancy enumeration (voxels)
  • Used in FEA meshing, additive manufacturing
Euler–Poincaré Validity Formula: V − E + F − H = 2(B − G) where V = vertices, E = edges, F = faces H = holes in faces (through-holes in faces) B = number of bodies G = genus (through-holes in body, e.g. G=1 for torus) Example — Cube: V=8, E=12, F=6, H=0, B=1, G=0 8 − 12 + 6 − 0 = 2(1−0) = 2 ✓
Unit 04

04. Computer Aided Engineering (CAE)

🔬 Unit Overview

Covers the complete simulation-driven engineering analysis workflow — from importing geometry and defining material models through meshing, applying loads and boundary conditions, running FEA solvers, post-processing results, and validating against physical tests — with a case study in ANSYS.

CAE Workflow

Step 01

Pre-processing

Import CAD geometry (IGES/STEP), simplify for analysis, define coordinate systems.

Step 02

Material Definition

E, ν, ρ, yield strength, fatigue data; isotropic / orthotropic / non-linear models.

Step 03

Meshing

Element type selection, mesh density, refinement at stress concentrations; quality checks.

Step 04

Loads & BCs

Forces, pressures, moments; fixed supports, symmetry planes; thermal and inertial loads.

Step 05

Solver

Direct (sparse), iterative (CG); linear static, modal, transient, non-linear.

Step 06

Post-processing

Von Mises stress, deformation, safety factor, mode shapes, fatigue life contours.

Detailed Topics

  • Introduction to simulation — purpose, accuracy vs. cost trade-off, virtual prototyping
  • Interference detection — clash checking, clearance analysis in assemblies
  • Failure analysis under static loading — stress, displacement, buckling (linear)
  • Failure analysis under dynamic loading — modal analysis, harmonic response, transient FEA
  • Elements used in CAE
  • 1D: Beam (BEAM188), Spring, Truss elements
  • 2D: Plane stress/strain (PLANE182), Shell (SHELL181)
  • 3D: Tetrahedra (TET10), Hexahedral (HEX20, SOLID186)
  • Meshing — free meshing vs. mapped meshing; aspect ratio, Jacobian, skewness quality metrics
  • Optimal mesh — convergence study (h-refinement, p-refinement)
  • Loading and constraints; boundary conditions — fixed, pinned, roller; symmetry exploitation
  • Solvers — direct (LU factorisation), iterative (PCG); non-linear Newton-Raphson
  • Post-processing and reviewing results — stress contours, FOS, deformed shape animation
  • Validation of CAE results — comparison with analytical solutions, physical test data (correlation)
  • Case study using ANSYS software — bracket or connecting-rod static structural analysis
💡 Validation Principle

CAE results are only as trustworthy as the model's assumptions. Always validate against a closed-form solution or physical test before using results for design decisions — mesh independence study and material property accuracy are the two largest sources of error.

Unit 05

05. Computer Aided Manufacturing (CAM)

⚙️ Unit Overview

Traces the evolution from manual NC to modern CNC and DNC systems, examines flexible and automated manufacturing architectures (FMS, AGV, AS/RS), and introduces the digital planning and integration layers of CIM and CAPP.

NC / CNC / DNC Evolution

NC

Numerical Control

Punched tape; fixed program; no feedback loop; hard-wired controller. High setup time, no editing.

CNC

Computer Numerical Control

Dedicated onboard microcomputer; soft-wired; program stored in RAM; tool compensation, canned cycles, conversational programming.

DNC

Direct / Distributed Numerical Control

Central host computer distributes NC programs to multiple CNC machines over a network (RS-232 → Ethernet → MTConnect).

AC

Adaptive Control

Real-time sensor feedback (force, vibration, temperature) adjusts feeds/speeds in-process — geometric AC (ACC) and optimisation AC (ACO).

NC Advantages, Disadvantages & Features

NC/CNC Advantages

Why CNC?

  • High repeatability and accuracy
  • Complex 3D contours achievable
  • Reduced setup and inspection time
  • Lights-out manufacturing possible
  • Easy design change via program edit
NC Disadvantages

Limitations

  • High initial capital investment
  • Skilled programmers required
  • Not cost-effective for one-offs
  • Maintenance complexity
CNC Features

Key CNC Capabilities

  • Tool length / radius compensation
  • Canned cycles (drilling, tapping, boring)
  • Parametric / macro programming
  • Sub-programs and subroutines
  • DNC interface and remote diagnostics

Automated Manufacturing Systems

  • Flexible Manufacturing System (FMS)
  • Components — CNC machining centres, material handling, tool management, computer control
  • FMS equipment and control — cell controller, DNC, scheduler, real-time monitoring
  • Automated Guided Vehicle Systems (AGVS)
  • Wire-guided, laser-guided, vision-guided navigation
  • Load/unload, towing, unit-load, forklift types
  • Traffic management, blocking, collision avoidance
  • Automated Storage and Retrieval System (AS/RS)
  • Unit-load, mini-load, person-on-board, deep-lane types
  • Storage/retrieval machine (SRM), aisle structure, control system
  • Throughput analysis — single and dual command cycles
  • Computer Integrated Manufacturing (CIM) — integration of CAD/CAM/CAE/MRP/ERP
  • Computer Aided Process Planning (CAPP)
  • Variant (retrieval) CAPP — retrieve and modify standard plan
  • Generative CAPP — decision logic, GT codes, AI-based
